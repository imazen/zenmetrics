# JXL lossy P0 — first knob main-effects signal (2026-06-25)

First real `lossy_dense` sweep for the JXL lossy knob-space ablation program
([`../docs/JXL_LOSSY_KNOBSPACE_ABLATION_PROGRAM.md`](../docs/JXL_LOSSY_KNOBSPACE_ABLATION_PROGRAM.md)).
**Local-first** (per the picker-pipeline pattern — the Hetzner fleet is for *scale*;
this proves the pipeline + gives the first directional signal).

## Status: PRELIMINARY / question-discovery — NOT shippable calibration

- **n = 12 images** (9 photo / 3 screen), **≤1 MP**, **ssim2 only**, **lean max-dev 1** (each
  knob probed at e7; effort ladder e1–e9 at default knobs). This is Q0 *question-discovery* —
  which knobs to focus on vs deprioritize — **not** a calibration-grade verdict.
- Does NOT yet satisfy the house sweep discipline (≥50/content-class, multi-metric
  butteraugli+cvvdp, full tiny→large size range, line-art + mixed content). Those are the
  fleet-scale confirmation run.
- Lean caveat (the accepted P0 tradeoff): knobs are measured at **e7 only**, so a knob whose
  payoff is e9-only is invisible here. Revisit under effort×knob if the oracle gap points there.

## Method

`zenmetrics sweep --codec zenjxl --plan lossy_dense --q-grid 5,10,…,95 --metric ssim2 --jobs 4`
(481 cells/image, 5772 total, **0 encode/score failures**). Per-image **global Pareto front**
(ssim2 vs encoded bytes); each cell classified into its single deviation arm (the max-dev-1
structure). Data + analysis (block storage + Tower):
`/mnt/v/output/jxl-lossy-p0-2026-06-25/lossy_dense_p0.tsv`
(sha256 `30e2172c13a13449ab7e2388769dca5afe4d4a9ddca0eba9828e82b8732ad671`),
`p0_pareto_analysis.txt`.

## Findings (directional)

**Front contributors (share of 421 front points):** effort ladder `e9` 15.4%, `e3`–`e6` (photo-skewed);
`gaborish-off` 10%; the **`k_ac_quant` ladder** kaq0.88/1.0/0.65/0.575 ≈ 22% combined; `strategy`
lean 6.7% / libjxl 1.2%; `faster_decoding` `fd4` 4.3%.

**Content-dependence (the picker signal):**
- `strategy` lean/libjxl cost **~−7.5 ssim2 on screen vs ~−0.5 on photo** (matched-q) — strongly
  content-dependent; the shipping `zen` default is much better on screen content.
- effort ladder + `ans0` + the entropy_mul presets contribute to the **photo** front only.
- `fd4` / `exp` (Experimental mode) also photo-favorable, screen-unfavorable.

**Never on any front (dominated everywhere):** `e1`, `e2`, `prog1` (and `prog2` once).
