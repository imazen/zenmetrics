# T_y.B.1 (2026-05-17) — ssim2-gpu FIR D=5 vs IIR baseline

Hardware: AMD Ryzen 9 7950X (water-cooled, 128 GB), NVIDIA RTX 5070,
CUDA 13.2.1, cubecl 0.10.0 (lilith/cubecl fork de2f9857).
Workload: 4000×3000 = 12 MP synthetic RGB pair via `examples/ssim2_bench_warm`.
Each measurement is 5 iters after 2 warmup iters.

Commit: feat/ssim2-fir tip (this commit) — branch from master 2f531309.
Build: `cargo build --release -p ssim2-gpu --features cuda --no-default-features`.
Runtime: `cubecl-cuda` (no `-C target-cpu=native`).

## Wall time

| Path                          | IIR baseline | FIR D=5 | Speedup |
| ----------------------------- | -----------: | ------: | ------: |
| compute (cold)                |    58.8 ms   | 42.1 ms |   1.40× |
| compute_with_reference (warm) |    39.1 ms   | 29.2 ms |   1.34× |

Score: -73.011710 (IIR) → -73.672287 (FIR). Drift consistent with
Kanetaka et al. IWAIT 2026 Table 2 (D=5 SROCC 0.890387 vs libjxl
0.889297 — paper does not assert score-magnitude parity).

## Per-kernel (nsys, full bench run = 5+5+2+2 iters)

### Baseline (Charalampidis IIR + transpose)

```
blur_pass_kernel:           216 ms / 2088 inst /  103 us avg   (43.2%)
thread_sum_p4_kernel:       133 ms /  756 inst                  (26.6%)
transpose_kernel:            60 ms / 1440 inst /   41 us avg   (11.9%)
error_maps_kernel:           40 ms /  252 inst /  158 us avg    (8.0%)
pointwise_mul_kernel:        25 ms /  648 inst /   39 us avg    (4.9%)
linear_to_xyb_planar_kernel: 12 ms /  132 inst                  (2.4%)
```

### FIR D=5

```
thread_sum_p4_kernel:       133 ms /  756 inst                  (45.7%)
error_maps_kernel:           40 ms /  252 inst                 (13.7%)
blur_h_fir5_kernel:          35 ms / 1044 inst /   33 us avg   (11.9%)
blur_v_fir5_kernel:          32 ms / 1044 inst /   31 us avg   (11.0%)
pointwise_mul_kernel:        25 ms /  648 inst                   (8.7%)
linear_to_xyb_planar_kernel: 12 ms /  132 inst                   (4.1%)
```

Blur-only: 216 → 67 ms = **3.25× faster**.
Blur+transpose: 276 → 67 ms = **4.1× faster** on the blur stage.
End-to-end wall time gains less than that because reduction (thread_sum_p4)
and error_maps stay constant — they become a larger share of the
remaining time.

## Parity drift (parity_jpeg_corpus 256×256)

| q  | cpu IIR | gpu FIR | abs Δ  | rel %  |
| -: | ------: | ------: | ----:  | -----: |
|  1 |   1.24  | -16.16  | 17.40  | 1404%  |
|  5 | -10.45  | -30.57  | 20.13  |  193%  |
| 20 |  57.07  |  47.79  |  9.28  |   16%  |
| 45 |  68.68  |  62.13  |  6.55  |   10%  |
| 70 |  79.51  |  76.68  |  2.84  |    4%  |
| 90 |  90.89  |  89.93  |  0.96  |    1%  |

Monotonic in q. Largest gaps at heavy distortion where the FIR's
tighter effective support (radius 2) captures sharper local error
than the IIR's wider (~radius 5) Charalampidis impulse response.
SROCC across image pairs is preserved by construction per the
paper.
