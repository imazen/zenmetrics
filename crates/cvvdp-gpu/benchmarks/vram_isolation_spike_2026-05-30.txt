# cubecl CUDA per-stream VRAM isolation spike — task #152
#
# Provenance:
#   repo commit (@):  19bda4049f2b (task152 design branch, on top of master 36ec863d)
#   cubecl fork:      zenforks-cubecl-* 0.10.1 (crates.io), tree 970ad5b5
#   host:             water-cooled 7950X, RTX 3060 12 GiB, native CUDA (driver 596.21)
#   command:          cargo run --release -p cvvdp-gpu --features cuda,cubecl-types \
#                         --example vram_isolation_spike
#   sampling:         nvidia-smi --query-gpu=memory.used, sampled AFTER client.sync()
#                     (deferred-free queue drained) + 150ms settle. Every number measured.
#   note:             baseline ~2.8 GiB is other processes on the box; total 12 GiB.
#
# CONCLUSION: per-stream VRAM pool isolation is REAL. Dropping a context on its
# own explicit stream + memory_cleanup() + sync() returns exactly that context's
# pages to the driver, independent of (and not disturbing) another context's pool.
# Reclaim works cross-thread (explicit stream_id overrides cubecl thread-local).
# Sharing a pool page (the default-stream case) blocks reclaim until ALL occupants
# of the page drop — the hazard that makes a bare Drop on the shared stream
# non-ironclad.

== cubecl CUDA per-stream VRAM isolation spike (task #152) ==
   max_streams=128, each StreamId.value maps to its own pool

  baseline (no cubecl ctx)            used=  2848 MiB
  per-context target ~= 1536 MiB (24 x 64 MiB)

-- create context A (stream value=101), allocate --

-- create context B (stream value=202), allocate --

  [after drop+cleanup A              ] used=  5345 MiB  (Δbase +2497 MiB)

-- CONTROL: two small allocs share a page, drop one --
  [control: both small alive         ] used=  3073 MiB  (Δbase +225 MiB)
  [control: dropped 1of2 + cleanup   ] used=  3073 MiB  (Δbase +225 MiB)
  [control: dropped 2of2 + cleanup   ] used=  3009 MiB  (Δbase +161 MiB)

-- THREAD MOBILITY: alloc on worker thread, reclaim on main --
  [worker-thread alloc (stream 404)  ] used=  5312 MiB  (Δbase +2464 MiB)
  [main-thread reclaim of 404        ] used=  3008 MiB  (Δbase +160 MiB)
  cross-thread reclaim freed +2304 MiB (target ~1536) -> THREAD-MOBILE: explicit stream_id overrides thread-local
