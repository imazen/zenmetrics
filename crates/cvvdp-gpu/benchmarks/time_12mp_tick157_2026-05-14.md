# cvvdp-gpu — 12 MP timing snapshot (tick 157, 2026-05-14)

Run after tick 157 split `compute_dkl_planes` and
`compute_dkl_gauss_pyramid` into dispatch-only helpers
(`_dispatch_dkl_planes_gpu`, `_dispatch_gauss_pyramid_gpu`) so the
weber-pyramid call no longer pays for two wasted readbacks: 36 MB
of level-0 DKL planes (from `compute_dkl_planes`) + ~190 MB of
the full 3-channel Gaussian pyramid (from `compute_dkl_gauss_pyramid`).
Both prior calls used `let _ = ...` to discard the host Vec.

## Environment

- Host: lilith's water-cooled 7950X / 128 GB / RTX-class CUDA
- Build: `cargo run --release --example time_12mp -p cvvdp-gpu --features cuda`
- Synthetic 4000×3000 RGB8 pair

## What changed

Tick 157 (`pipeline.rs`):

1. `compute_dkl_planes` extracted `_dispatch_dkl_planes_gpu`: the
   color kernel launch lives in the helper, the public function
   wraps it with the 3-channel level-0 readback.

2. `compute_dkl_gauss_pyramid` extracted `_dispatch_gauss_pyramid_gpu`:
   `_dispatch_dkl_planes_gpu` + the per-level `downscale_kernel`
   reduce, no readback. The public function wraps it with the
   full-pyramid readback.

3. `_dispatch_weber_pyramid_gpu` now calls
   `_dispatch_gauss_pyramid_gpu` directly — was calling the public
   `compute_dkl_gauss_pyramid` with `let _ = ...`, paying for a
   ~190 MB host transfer that it discarded.

4. `compute_dkl_laplacian_pyramid` same swap (dispatch-only path).

Combined effect per weber call: ~230 MB host transfer skipped (36
MB level-0 + ~190 MB pyramid), times 2 weber calls per JOD.

## Open observations

- The 2×weber + bucket fits inside d_bands now (negative bucket
  in the wide-noise sense), confirming the weber dispatch + the
  three reduce launches per level + downstream CSF/mask/IO are
  bandwidth-tightly packed.
- Next big target: the weber pyramid itself, where the
  upscale-then-subtract bandwidth dominates. Candidates:
  - Shared-memory tiled upscale for the L_bkg / per-channel expand
    step (each thread block could load a coarse tile once and
    serve multiple destination pixels).
  - Fuse upscale_v + upscale_h into a single launch with a small
    private-memory ring buffer (saves the intermediate `vscratch`
    write+read).
  - Move the baseband mean reduction to GPU (currently 4 small
    readbacks + 3 reuploads per side; small in bytes but each is
    a synchronous drain).
- Medium→large size sweep needs re-measurement now that almost
  all wasted-host overhead is gone.
