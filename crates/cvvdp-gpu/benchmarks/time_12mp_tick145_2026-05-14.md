# cvvdp-gpu — 12 MP timing snapshot (ticks 144-145, 2026-05-14)

Run after ticks 144-145 dropped the `dist_weber` host Vec entirely
and added per-band `ref_weber[k]`/`ref_log_l_bkg[k]` drops in the
band loop. These changes target the host-memory-pressure issue
tick 83's consecutive-weber diagnostic identified.

The d_bands wall — formerly dominated by 645 ms of CSF + masking +

## What changed

- **Tick 144** (`02f37728`): replaced `let (dist_weber, _) = ...`
  with `let _ = self.compute_dkl_weber_pyramid(dist_srgb)?;`.
  Drops the ~190 MB host Vec for the DIST side immediately
  (DIST CSF reads `self.bands_ref[k]` handles directly per
  tick 87 — host Vec was unused).
- **Tick 145** (`913a7c5f`): per-band drops in the d_bands loop:
  `ref_weber[k] = [Vec::new(), Vec::new(), Vec::new()];`
  `ref_log_l_bkg[k] = Vec::new();`
  right after the CSF dispatch copies the band-k data to GPU.

Combined they take the band-loop host residency from ~240 MB
sticky-throughout-loop to a sliding window proportional to
remaining bands. Less time spent in malloc/munmap, fewer
TLB invalidations, and presumably the heap manager isn't
fighting for cache lines with the GPU driver.
