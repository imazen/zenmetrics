# cvvdp-gpu — 12 MP timing snapshot (tick 158, 2026-05-14)

Run after tick 158 added `baseband_divide_3ch_kernel` so the
weber-pyramid baseband path no longer reads back +
re-uploads the 3 coarsest Gaussian planes. The host still
reads back `gauss_a` once (small buffer, ~744 pixels at
MAX_LEVELS=8 / 12 MP) to compute `l_bkg_mean`; the per-channel
division now happens on GPU using the host-computed mean as a
scalar uniform.

## What changed

Tick 158 adds `baseband_divide_3ch_kernel` in `pyramid.rs` and
wires it into `_dispatch_weber_pyramid_gpu` (pipeline.rs). The
old baseband path:

```
for c in 0..3 {
    let g = gauss_ref[last].planes[c].clone();
    let bytes = client.read_one(g)?;      // SYNC drain
    let divided: Vec<f32> = ...;
    let h = client.create_from_slice(...); // upload
    bands_dest[last].planes[c] = h;
}
```

becomes:

```
let inv_mean = 1.0 / l_bkg_mean;
baseband_divide_3ch_kernel::launch::<R>(
    // gauss[last][0..3] in, bands_dest[last].planes[0..3] out
    // inv_mean as scalar uniform
);
```

Net change per weber call: 3 readbacks + 3 reuploads (6 GPU sync
drains, ~9 KB each at MAX_LEVELS=8 / 12 MP baseband) become 1
GPU launch writing into pre-allocated handles. The
`bytes_a`/`l_bkg_mean` readback is unchanged — still needed for
the host-side `log_l_bkg_baseband` return value.

The win is small in wall-time at 12 MP (~1 ms of host roundtrip
overhead spread across 2 weber calls per JOD) because the
baseband is tiny. The structural benefit is bigger: weber
pyramid now has 1 sync drain (the gauss_a readback for the
mean) per side instead of 4.

## Open observations

- 4 sync drains per weber → 1. Bandwidth saved is small at the
  baseband level (it's the coarsest, smallest pyramid level), but
  the drain count is what mattered.
- Next big target: still the weber pyramid's 6 upscale launches
  per level (3 channels × {upscale_v, upscale_h}) and the
  intermediate vscratch buffers. Fusing upscale_v + upscale_h
  into a single launch (private-memory ring buffer) would halve
  launch count and skip an n_v-sized buffer round-trip per
  channel per level.
- The L_bkg expand (one extra upscale_v + upscale_h per level
  for the shared luminance) is a smaller version of the same
  problem.
- 1 sync drain is the lower bound for this version of the code
  (the host needs `l_bkg_mean.log10()` to return); pushing it to
  zero would require pushing the entire `log_l_bkg_baseband`
  scalar through the pipeline as a tiny GPU buffer all the way
  to the pool step.
