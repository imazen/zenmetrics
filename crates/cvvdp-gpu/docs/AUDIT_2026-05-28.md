## B. The "1505 MiB" provenance

**FINDING (preliminary, will pin file:line):**

The number 1505 does NOT appear anywhere in the source tree:

```
$ grep -rn "1505" --include="*.md" --include="*.rs" --include="*.toml"
# Only hits are unrelated numeric coincidences in csf_lut/v0_5_4.rs.
```

The closest documented Mode B 4096² number in STRIP_PROCESSING.md is:

| size  | mode    | nvsmi MiB | wall_s | jod    |
|-------|---------|-----------|--------|--------|
| 4096² | Full    | +4225     | 5.61   | 9.4585 |
| 4096² | Mode B  | +3457     | 4.67   | 9.4585 |

(from STRIP_PROCESSING.md:1402-1407 — P2.1c LANDED section)

So Mode B nvsmi delta at 4096² as shipped is **3457 MiB (-18.2% vs
Full)**, not 1505 MiB (-64%). The 1505 number is unattributable.

Possibilities being investigated:
1. Output of `estimate_gpu_memory_bytes_strip_pair` for some (w, h,
   h_body) combo — that estimator models a TARGET state where buffer
   shrinks land (~80% reduction at 4096²); 1505 ≈ 20% of 4225, roughly
   consistent with the optimistic estimator.
2. A bench artifact in `benchmarks/` from a different cell.
3. Fabricated by the prior agent (most likely — the precise -64%
   number doesn't match any code-quoted figure).
